# conversation chart

Paper-figure oriented diagram skill with a **mandatory local file/script pipeline**, multi-round controlled edits, and source-to-visual traceability.

![hero](hero.png)

## Mandatory Local Interface

This project treats local interface as a hard requirement:

- Inputs: `input.md`, `spec.json`, `chart.dsl.json`
- Optional plan input: `color-plan.json` (LLM generated)
- Outputs: `output.png`, `output.layout.json`, `output.occlusion.json`, `output.map.json`, `output.svg`, `output.tex`
- Optional outputs: `output.pdf`, `output.from_tex.png`
- Core scripts:
  - `scripts/run_local_pipeline.py`
  - `scripts/apply_color_plan.py`
  - `scripts/render_png.py`
  - `scripts/analyze_occlusion.py`
  - `scripts/render_svg.py`
  - `scripts/render_tikz.py`
  - `scripts/render_tikz_chain.py`

## Quick Start

Run local pipeline from `spec.json`:

```bash
python scripts/run_local_pipeline.py --spec examples/local_interface/spec.json
```

Generated files (example: `outputs/local-interface-demo/`):

- `output.png`
- `output.layout.json`
- `output.occlusion.json`
- `output.map.json`
- `output.svg`
- `output.tex`
- `output.manifest.json`
- optional: `output.colored.dsl.json`, `output.pdf`, `output.from_tex.png`

## Occlusion-Aware Rendering

`render_png.py` + `analyze_occlusion.py` jointly provide:

- orthogonal obstacle-avoid routing
- edge-vs-node-text crossing checks
- edge label overlap checks
- edge label placement conflict minimization
- quality gate via `max_occlusion_issues`

`run_local_pipeline.py` status:

- `status = ok`: all checks pass
- `status = warning`: occlusion gate exceeded
- `status = failed`: render/export step failed

## Category-Based Color Control (Skill/LLM Driven)

Color strategy is generated by skill/LLM into `color-plan.json`. Renderer only executes the plan.

`spec.json` extension:

```json
{
  "dsl": "chart.dsl.json",
  "color_plan": "color-plan.json"
}
```

`color-plan.json` example:

```json
{
  "color_mode": "category",
  "category_source": "group",
  "category_assignments": {
    "n_user": "access",
    "n_gateway": "access",
    "n_retriever": "retrieval"
  },
  "category_colors": {
    "access": "#56B4E9",
    "retrieval": "#009E73",
    "generation": "#E69F00",
    "ops": "#CC79A7"
  },
  "theme_overrides": {
    "background": "#F8FAFC",
    "edge_color": "#475569"
  }
}
```

Pipeline applies it via `scripts/apply_color_plan.py` and renders from `*.colored.dsl.json`.

Sample plan file: `examples/local_interface/color-plan.sample.json`

## TikZ + SVG Strategy

- TikZ/LaTeX chain: high-quality backend for paper figures
  - `tectonic` or `pdflatex`
  - optional PDF->PNG conversion via `pdftoppm` / `magick`
  - compile log + status JSON output
- SVG backend: compatibility for Figma/Illustrator/web/README

Note: user-facing value is paper-figure generation, not a TikZ-only tool.
