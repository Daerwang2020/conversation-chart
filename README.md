鬠ｿ# :sparkles: Conversation Chart :bar_chart:

A paper-figure diagram skill for **Claude/Codex workflows**, built on a **mandatory local file/script pipeline**, **controllable multi-round edits**, and **source-to-visual traceability**.

![hero](assets/demo.jpg.png)

## Why it stands out

- **Local-first, reproducible**: every figure goes through explicit files and scripts
- **Paper-oriented outputs**: optimized for architecture, workflow, and system diagrams
- **Controllable edits**: supports multi-round visual refinement instead of one-shot generation
- **Traceable artifacts**: layout, occlusion, mapping, SVG, and TikZ outputs are preserved

## Mandatory Local Interface

**Inputs**
- `input.md`
- `spec.json`
- `chart.dsl.json`

**Optional input**
- `color-plan.json` (generated by skill/LLM)

**Outputs**
- `output.png`
- `output.layout.json`

- `output.map.json`
- `output.svg`

**Optional outputs**

- `scripts/apply_color_plan.py`
- `scripts/render_png.py`
- `scripts/analyze_occlusion.py`
- `scripts/render_svg.py`
- `scripts/render_tikz.py`
- `scripts/render_tikz_chain.py`

## Quick Start

```bash
python scripts/run_local_pipeline.py --spec examples/local_interface/spec.json
```

Generates the output files listed above in `outputs/local-interface-demo/`, including `output.manifest.json` and optional files like `output.colored.dsl.json`, `output.pdf`, `output.from_tex.png`.

## Occlusion-Aware Rendering

`render_png.py` + `analyze_occlusion.py` provide:

- orthogonal obstacle-avoid routing
- edge-vs-node-text crossing checks
- edge label overlap checks
- edge label placement conflict minimization
- quality gating via `max_occlusion_issues`

Pipeline status:

- `ok`: all checks pass
- `warning`: occlusion gate exceeded
- `failed`: render/export failed

## Category-Based Color Control

Color strategy is generated into `color-plan.json`; the renderer only executes the plan.

`spec.json` extension:

```json
{
  "dsl": "chart.dsl.json",
  "color_plan": "color-plan.json"
}
```

`color-plan.json` example:

```json
{
  "color_mode": "category",
  "category_source": "group",
  "category_assignments": {
    "n_user": "access",
    "n_gateway": "access",
    "n_retriever": "retrieval"
  },
  "category_colors": {
    "access": "#56B4E9",
    "retrieval": "#009E73",
    "generation": "#E69F00",
    "ops": "#CC79A7"
  },
  "theme_overrides": {
    "background": "#F8FAFC",
    "edge_color": "#475569"
  }
}
```

Pipeline applies it via `scripts/apply_color_plan.py`.

Sample plan file: `examples/local_interface/color-plan.sample.json`
