# Output Contract

## Local Interface

Minimum required local inputs/outputs:

- Inputs:
  - `input.md`
  - `spec.json`
  - `chart.dsl.json`
  - optional: `color-plan.json`
- Outputs:
  - `output.png`
  - `output.layout.json`
  - `output.occlusion.json`
  - `output.map.json`
  - `output.svg`
  - `output.tex`
  - optional: `output.pdf`, `output.from_tex.png`

## `spec.json` Schema (pipeline entry)

```json
{
  "dsl": "path/to/chart.dsl.json",
  "color_plan": "path/to/color-plan.json",
  "input_md": "input.md",
  "output_dir": "outputs/run-001",
  "basename": "output",
  "formats": ["png", "svg", "tex", "pdf"],
  "tikz_compile": true,
  "tikz_engine": "auto",
  "max_occlusion_issues": 0
}
```

If `color_plan` is provided, pipeline executes:

```bash
python scripts/apply_color_plan.py --dsl <dsl> --plan <color_plan> --output <basename>.colored.dsl.json
```

Then downstream render/export uses the colored DSL.

## `color-plan.json` Schema (LLM output)

```json
{
  "color_mode": "category",
  "category_source": "group",
  "category_assignments": {
    "n_user": "access",
    "n_retriever": "retrieval"
  },
  "category_colors": {
    "access": "#56B4E9",
    "retrieval": "#009E73"
  },
  "node_cycle": ["#56B4E9", "#009E73", "#E69F00"],
  "theme_overrides": {
    "background": "#F8FAFC",
    "edge_color": "#475569"
  }
}
```

## `chart.dsl.json` Schema (source of truth)

Top-level:

```json
{
  "version": "1.2",
  "direction": "LR",
  "theme": {
    "background": "#F8FAFC",
    "canvas_bg": "#F8FAFC",
    "color_mode": "category",
    "category_source": "group",
    "category_colors": {"retrieval": "#009E73"},
    "category_assignments": {"n_retriever": "retrieval"},
    "node_cycle": ["#56B4E9", "#E69F00", "#009E73"]
  },
  "groups": [],
  "nodes": [],
  "edges": []
}
```

Theme notes:

- `color_mode`: `category` (recommended) or `node`
- `category_source`: `auto`, `group`, `node`, `none`
- `category_colors`: category -> hex color mapping
- `category_assignments`: node id -> category mapping (LLM output)
- `node_cycle`: optional fallback cycle when category has no explicit color
- `background` and `canvas_bg` are aliases (either is accepted)

Node:

```json
{
  "id": "n_retriever",
  "label": "Retriever",
  "x": 560,
  "y": 430,
  "width": 220,
  "height": 88,
  "shape": "rounded",
  "style": {"fill": "#ECFCCB", "stroke": "#65A30D", "text": "#365314"}
}
```

Edge:

```json
{
  "id": "e_12",
  "from": "n_query",
  "to": "n_retriever",
  "label": "Retrieve",
  "relation": "sync_request",
  "style": {
    "route": "orthogonal",
    "pattern": "solid",
    "direction": "single",
    "width": 3,
    "color": "#2563EB",
    "avoid_obstacles": true
  }
}
```

## `output.layout.json` Schema

Generated by `render_png.py`:

- `canvas`
- `direction`
- `theme`
- `groups` (with `bbox_px`)
- `nodes` (with `bbox_px`, `text_bbox_px`)
- `edges` (with `polyline_px`, `label_bbox_px`, style, relation)
- `occlusion`

## `output.occlusion.json` Schema

Generated by `analyze_occlusion.py`:

```json
{
  "layout_file": "output.layout.json",
  "issue_count": 0,
  "status": "ok",
  "issues": []
}
```

## Validation Rules

- IDs must be unique and stable across rounds.
- Edge endpoints must reference existing nodes.
- `issue_count` should satisfy `issue_count <= max_occlusion_issues`.

## `output.manifest.json` Status

`run_local_pipeline.py` writes pipeline status for CI/release checks:

- `status = "ok"`: all required steps succeeded and occlusion gate passed
- `status = "warning"`: rendering succeeded, but occlusion gate failed
- `status = "failed"`: at least one render/export/compile step failed

Manifest additionally records:

- `color_plan` (resolved path or null)
- `effective_dsl` (source DSL or generated `*.colored.dsl.json`)
- `outputs.colored_dsl` (when color plan is applied)

Exit code contract:

- `0`: ok
- `2`: warning (occlusion gate exceeded)
- `3`: failed
